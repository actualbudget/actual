# Grouping Transactions

The account register can group transactions into collapsible sections while keeping the transactions editable in place.

Use **Group by** in the transaction toolbar to choose one of the available fields:

- Date: day, week, month, or year
- Category
- Category group
- Payee
- Account, in multi-account views
- Cleared status

Each group header shows the group label, the number of transactions in the group, and the sum of the grouped transactions. Click the group header to expand or collapse that section. When grouping is active, the account menu also includes **Expand all groups** and **Collapse all groups**.

Filtering and sorting still apply to the transaction list. Group totals are calculated from the visible filtered transactions, not from transactions hidden by the current filter. Sorting controls the order of transactions inside each group.

Running balances are hidden while grouping is active because a running account balance is only meaningful in a single chronological list. Select **Group by: None** to return to the normal transaction list and show running balances again.

Saved filters remember the selected grouping option. If you save or update a filter while grouping is active, reloading that saved filter will restore the same grouping.
